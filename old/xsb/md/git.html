git的使用

git push git commit 主要使用的就是这几个 - - - - 主要是基于已有的代码 Git鼓励大量使用分支：

  • 查看分支：git branch
  • 创建分支：git branch
  • 切换分支：git checkout
  • 创建+切换分支：git checkout -b
  • 合并某分支到当前分支：git merge
  • 删除分支：git branch -d

git的使用总结

  • 开发项目 先进行fork 在merge request 在mentor阅读源码发现源码ok后,就可以merge。
  • 对文件进行修改之后，需要git commit -a -m “xxxx” 这样才可以一次性将多个文件一起提交（或者 git add 所有修改过的文件－－ 包括添加，删除的文件和修改的文件，再去git commit -m "commit info"）

git 回滚

git reset --hard 例子如下 git reset --hard c90dfaf7d7cb1f05dd5389255d93a0fd8efc46f7 git push origin HEAD --force

上面是先将本地的代码回滚到每一个版本，再去push覆盖线上版本

删除撤销

  • rm删除 通过 gc — xx.xx 进行恢复（gc -> git checkout） （被删除的文件还在暂存区域）
  • git rm 删除 gc 无效 只能通过git reset — hard commitid 进行恢复
